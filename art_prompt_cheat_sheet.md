
- Add the following keywords to the 'positive_prompt' for all retro pixel character art:

solid white background, no gradient backgrounds, pure white, no shadows, crisp edges

Add the following keywords to the 'negative_prompt' for all retro pixel character art:

gradient backgrounds, colored backgrounds, shadows, blurry edges, varied lighting
- Add the following to the 'positive_prompt' for all character sprites and icons:
  - full body
  - centered
  - entirely inside frame 
  - zoomed out

- Add the following to the 'negative_prompt' for all character sprites and icons:
  - cropped
  - out of frame
  - close-up
  - portrait
- Add "camera angle, perspective" to the 'positive_prompt' for all future AI-generated art in Unity games.
For top-down view games or perspectives (like many strategy games), also add:
  - "top-down perspective, birds-eye view"
to the positive prompt.
For side-scrolling platformer games with flat 2D profiles, include:
  - "side-scroller perspective, flat 2d profile" 
in your positive prompt as well. This will ensure clarity on the intended game art style and viewpoint for the AI to correctly generate assets in line with the Master GDD camera design specifications.
- Add the following keywords to the 'negative_prompt' for every generated asset:
  - text
  - watermark
  - signature 
  - UI elements
  - health bars
  - letters
  - words
- Add the following keywords to the 'positive_prompt' in future:
  - flat landscape 
  - wide angle
  - gameplay area
  - level design
  - clear pathways

- Add the following keywords to the 'negative_prompt' in future:  
  - characters
  - entities
  - players
  - UI
  - foreground objects

These rules ensure that when generating AI art for environment maps and backgrounds, the output will emphasize an empty, wide-angle view suitable for level design and gameplay, without including elements that would clutter or distract from the intended focus area.
